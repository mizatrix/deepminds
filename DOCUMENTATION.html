FOR USERS Getting Started

What is the Academic Excellence Portal?

The Academic Excellence Portal is a comprehensive platform for tracking and showcasing student achievements across various categories including scientific research, competitions, hackathons, sports, volunteering, and more.

Account Types

Account Type Features
Student Account Submit achievements, View leaderboard, Earn points and badges, Download certificates, Track progress
Admin Account Review submissions, Approve/reject achievements, Manage students, View analytics, Configure system settings
ğŸ‘‘ Super Admin Account All Admin features PLUS: Promote/demote admins, Create new admins, Delete admin users, Cannot be demoted or deleted. Only 1 Super Admin allowed.

Creating an Account

  1. Navigate to Registration: Click "Join Now" on the homepage or visit /register
  2. Fill Registration Form:
    • Email address (required)
    • Full name (required)
    • Password (min 8 characters, must include uppercase, lowercase, number, special character)
    • Confirm password
  3. Submit: You'll be redirected to login page
  4. Alternative - Social Login: Click "Continue with Google/GitHub/Microsoft"
ğŸ’¡ First Login: After your first login, you'll see a Profile Completion Banner. Complete your profile to unlock all features!

Editing Submissions

Only pending submissions can be edited. Approved and rejected submissions are permanently locked.

Edit Process:

  1. Click the Edit button (pencil icon) on a pending submission
  2. Confirm: "Are you sure you want to edit this submission?"
  3. Edit modal opens with pre-filled form
  4. Modify any fields: Title, Category, Organization, Location, Date, Description
  5. Click "Save Changes"
  6. Confirm: "Are you sure you want to save these changes?"
  7. Submission updated and list refreshes automatically
âš ï¸ Important: Evidence files cannot be changed after initial submission. Only pending submissions can be edited. No submissions can be deleted to ensure data integrity.

Submitting an Achievement

  1. Navigate to Dashboard (/student/dashboard)
  2. Fill Achievement Form:
    • Title: Name of your achievement
    • Category: Select from dropdown (Scientific, Artistic, Competition, etc.)
    • Organization: Where you achieved this
    • Location: City/Country
    • Date: When you achieved this
    • Description: Detailed explanation (max 500 characters)
    • Evidence: Upload proof (PDF, image, max 10MB)
  3. Click "Submit Achievement"

FOR USERS Frequently Asked Questions

General

Q: How do I reset my password?
A: Click "Forgot Password" on login page, enter your email, and follow the reset link.

Q: How long does submission review take?
A: Typically 24-48 hours. You'll be notified when reviewed.

For Students

Q: Why was my achievement rejected?
A: Check the admin feedback on your submission. Common reasons: insufficient evidence, doesn't meet criteria, duplicate submission.

Q: Can I edit a submitted achievement?
A: Yes, but only if it's still "Pending". Once approved or rejected, submissions cannot be edited. You'll need to click the Edit button and confirm twice (once to open the editor, once to save changes).

Q: Can I delete a submission?
A: No, submissions are permanent once created. This ensures data integrity and prevents accidental loss of achievement records.

Q: How do I earn more points?
A: Submit achievements in high-value categories (Scientific Research, Entrepreneurship, Hackathons = 100 points each).

Q: What happens to my profile completion percentage?
A: It updates automatically as you fill in required fields. Students need 8 fields (100%), admins need 6 fields (100%). The completion banner will disappear once you reach 100%.

FOR DEVELOPERS Database Schema

Users Table

model User {
  id            String   @id @default(cuid())
  email         String   @unique
  name          String
  password      String?
  role          String   @default("STUDENT")
  
  // Profile fields
  avatar        String?
  faculty       String?  // Students
  year          String?  // Students
  studentId     String?  // Students
  department    String?  // Admins
  position      String?  // Admins
  phone         String?
  bio           String?
  
  // Social links
  linkedIn      String?
  github        String?
  twitter       String?
  website       String?
  
  // Completion tracking
  profileCompleted      Boolean @default(false)
  completionPercentage  Int     @default(0)
  
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

  title            String

  achievementDate  String
  description      String
  evidenceUrl      String
  evidenceFileName String?
  evidenceFileType String?
  status           String   @default("PENDING")
  points           Int?
  submittedAt      DateTime @default(now())
  reviewedAt       DateTime?
  reviewedBy       String?
  adminFeedback    String?
}

AuditLog Table

model AuditLog {
  id              String   @id @default(cuid())
  timestamp       DateTime @default(now())
  action          String   // submit, approve, reject, delete
  performedBy     String
  performedByName String
  submissionId    String
  submissionTitle String
  details         String?
}

Notification Table (NEW in v2.2.0)

model Notification {
  id        String   @id @default(cuid())
  userId    String   // Email of the recipient
  title     String
  message   String
  type      String   @default("INFO") 
            // Types: INFO, SUCCESS, ERROR, 
            // SUBMISSION_APPROVED, SUBMISSION_REJECTED,
            // BADGE_EARNED, CERTIFICATE_ISSUED
  read      Boolean  @default(false)
  link      String?  // Optional deep link
  createdAt DateTime @default(now())

  @@index([userId, read])
  @@index([userId, createdAt])
}
